💡 ROUTE CREATION TIPS
═══════════════════════════════════════════════════════════════════════════

ADD NEW ROUTE:
1. Choose appropriate route group
2. Use RESTful URL patterns
3. Apply authentication filter if needed
4. Use parameter validation (:num, :any)
5. Test the route thoroughly

ROUTE GROUP STRUCTURE:
$routes->group('prefix', ['filter' => 'auth'], function($routes) {
    $routes->get('path', 'Controller::method');
    $routes->post('path', 'Controller::method');
});

PARAMETER TYPES:
├─ (:num) - Numeric only (e.g., user ID)
├─ (:any) - Any string (e.g., category name)
└─ (:segment) - URL segment (e.g., slug)

FILTERS:
├─ ['filter' => 'auth'] - Requires authentication
├─ ['filter' => 'role:admin'] - Requires specific role
└─ ['filter' => 'permission:manage_users'] - Requires specific permission

📍 QUICK DEBUGGING
═══════════════════════════════════════════════════════════════════════════

If route not working:
1. Check route syntax in Routes.php
2. Verify controller and method exist
3. Check authentication requirements
4. Test with different user roles
5. Check URL parameters

If 404 error:
1. Verify route is defined
2. Check URL spelling
3. Verify parameter types
4. Check route group structure
5. Test with different HTTP methods

If authentication error:
1. Check if user is logged in
2. Verify user role permissions
3. Check filter configuration
4. Test with different user types
5. Verify session data
